Arbiter — Multi-Agent AI Code Reviewer

A team of AI agents reviews every pull request — grounded in your codebase.

Arbiter is a GitHub App that reviews each PR the moment it opens: three specialist agents run in parallel, a supervisor synthesizes one verdict, and it posts straight to the PR — with context retrieved from the rest of your repository.

🧠

Multi-agent orchestration

Three specialist reviewers — code quality, test coverage, documentation — run in true parallel via LangGraph, then a supervisor merges their notes into one verdict.

LangGraph · fan-out / fan-in
🔍

Repo-aware reviews (RAG)

The repository is embedded into pgvector; each diff retrieves the most relevant code so agents judge changes against your actual codebase — not the diff in isolation. Incremental indexing re-embeds only what changed.

pgvector · Gemini embeddings
🔌

MCP server

The same reviewers are exposed as Model Context Protocol tools, so Claude Desktop, Cursor, or any MCP client can review a local diff — no webhook or GitHub App required.

Model Context Protocol
📋

Linear sync

When a PR maps to a Linear issue, the verdict is mirrored there as a comment and an arbiter: label — closing the loop back to where the work is tracked.

Linear GraphQL API
